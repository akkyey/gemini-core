# Design Validator Skill

GEMINI.md 第2章「Design First Protocol」および第1章「スキルの宣言要件」に基づき、品質と透明性を保証するための自己監査を行います。

## 監査項目

### 1. 設計の完全性 (Design Integrity)
実装前、またはセッション終了（wq）時に以下の存在を確認します。
- `docs/designs/` 内に今回のタスクに対応する設計メモがあるか。
- `docs/decision_log.md` (ADR) に今回の意思決定が追記されているか。

### 2. 透明性の確保 (Transparency Check)
直近の作業において、以下の宣言が適切に行われているか自己確認します。
- 🤖 **【スキル発動】 `skill_name`**
- ワークフロー実行開始時の対象プロジェクト宣言。

### 3. 整合性の維持 (Consistency Check)
- 各プロジェクトの `GEMINI.md` が正しく正典（gemini-core）を参照しているか。
- ワークフロー（start/wq）内のパスが現行環境と一致しているか。

## 運用タイミング
- **実装フェーズへの移行直前**: 設計の承認漏れを防ぎます。
- **`/wq` ワークフローの冒頭**: 締めくくり前の最終点検として実行します。

## 完了の定義
- [ ] すべての項目において「合格」または「要修正」の判断が下されていること。
- [ ] 不足がある場合、速やかに修正案をユーザーに提示していること。
