# /write-qiita-blog ワークフロー

このワークフローは、指定された情報源やテーマに基づいて、`qiita_stf` スキルのレギュレーション（整合性基準設計、6章構成）に従ったQiita向けブログ記事のドラフトを作成します。

## 実行手順

1. ユーザーから提供された情報源（ソースコード、過去のドキュメント、エラーログ等）やテーマの指示を確認する。
2. 絶対パス `/home/irom/dev/project-stock2/.gemini/.agent/skills/qiita_stf/SKILL.md` を読み込み、最新の `qiita_stf` スキルのルールとレギュレーションをコンテキストにロードする。
3. 提供された情報を元に、`qiita_stf` の「整合性基準設計」に合致するように内容を再構成し、以下の6章構成のドラフトを作成する。
    - 第1章：設計仮定の明示
    - 第2章：観測された不整合
    - 第3章：仮定の破綻
    - 第4章：再定義（設計転換）
    - 第5章：実装原理（核心のみ）
    - 第6章：結果と帰結
4. トーン（禁止語の不使用、推奨語の使用）をチェックし、ルールに沿うよう推敲する。
5. 作成したドラフトを `/home/irom/dev/gemini-docs/projects/project-stock2/blog/drafts/` ディレクトリ配下に、適切なファイル名（例：`YYYY-MM-DD_theme_name.md`）で保存する。
6. 保存したファイルのパスとともにユーザーに完了を報告し、レビューを依頼する。
