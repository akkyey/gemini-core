# `brain-review` ワークフロー (v24.2 Optimized)

本番の Markdown ファイルを直接編集せず、Brain（アーティファクト）内で全文プレビューによる推敲・レビュー・承認を経てからマージするための標準プロセスです。

> [!IMPORTANT]
> このワークフローは **一級市民のレビュー体験** を提供します。
> 単なる肯定的なフィードバック（「OK」「いいですね」等）だけでは本番反映は行われません。
> AI から「本番へ反映しますか？」と聞かれた後、**明示的に反映を許可（`/brain-review end` または「適用して」と具体的に指示）** するまでドラフト状態が維持されます。

## 起動コマンド (Commands)

### 1. 修正の開始: `/brain-review start [path]`
- **引数 `[path]`**: 修正対象の Markdown ファイルのパスを指定します。
- **排他制御（重要）**: すでに別のファイルが Brain でレビュー中の場合、新しいリクエストは即座に **却下 (Reject)** してください。ユーザーに現在のレビューを完了（`/brain-review end`）させるか、中断するように伝えます。一度にレビューできるファイルは **1つだけ** です。
- **ドラフトの展開**: 対象の全文を `proposed_{filename}.md` に展開し、`IsArtifact: true` で作成します。

### 2. 反映と終了: `/brain-review end`
- **内容の反映**: 現在 Brain 上で展開されているドラフトの内容で、本番のターゲットファイルを上書き更新します。
- **反映タイミング**: ユーザーからの**明示的な実行指示（コマンド）**があるまで、AI はドラフトの推敲を継続し、本番ファイルへの書き込みは行いません。
- **リポジトリ対応**: ターゲットファイルがシンボリックリンクや別リポジトリ（`gemini-docs` 等）にある場合、`git -C [repository_root]` を使用して変更を確定（Git コミット）します。
- **後片付け**: 本番への反映完了後、Brain 上の一時ファイル（ドラフト）を削除します。

## 実行ステップ

### Step 1: ドラフトの作成 (Start)
1. 対象ファイルの実パス（シンボリックリンクを解決したもの）を確認します。
2. 全文をコピーし、`/brain/` 配下に新規アーティファクトを作成します。
3. **重要**: `ArtifactMetadata` を設定し、`notify_user` を通じてシステム標準のレビュー画面をユーザーに提示します。

### Step 2: レビューと推敲 (Review)
1. ユーザーはサイドバーのレビューUIやチャットでフィードバックを行います。
2. AI は指示に従いファイルを更新し、再提示（`notify_user`）します。

### Step 3: 本番への書き戻し (End)
1. `/brain-review end` を受け取ったら、以下の手順を実行します：
   - a. `$ git -C [target_repo] add [file]`
   - b. `$ git -C [target_repo] commit -m "docs: finalize brain-review for [filename]"`
   - c. ドラフトの内容で本番ファイルを上書き。
2. Brain 上の一時ファイルを削除し、完了を報告します。

## 完了の定義
- [x] 指定された Markdown ファイルが **Review UI** を通じて全文提示されていること
- [x] ターゲットファイルのリポジトリ境界を越えて Git コミットが完遂されていること
- [x] `end` コマンドにより本番へ正確に反映され、一時ファイルが消去されていること
